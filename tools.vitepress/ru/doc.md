## Как пользоваться

1. Создайте задачу
2. Уточните у исполнителя, сколько времени ему потребуется на ее выполнение
3. Задайте дату фактического начала выполнения
4. Задайте дату прогноза со слов исполнителя (если требуется, возьмите таймаут и вернитесь к этому вопрсу позже)
5. Уточните у исполнителя, какова его оценка сложности задачи от 1 до 5 (либо возьмите таймаут и вернитесь к этому вопрсу позже)
6. Если потребовалась доработка, добавьте дополнительное время

::: warning
Если задачу невозможно оценить - лучше пока не оценивайте. Если задача переходит в качество особо непредсказуемых - ставьте оценку 6 (эти задачи не будут учитываться при анализе стандартных задач)
:::

## Про Business time

Это настройка для следующих целей:
1. Расчет затраченных бизнес-часов, потраченных на задачу (или ргуппу задач). В заданном формате нужно указать рабочие часы (поле `_descr` является опинальным)
2. Анализ эффективности рабочего времени

## Анализ затраченных бизнес-часов

Ожидается следующий формат для описания распорядка дня:
- `start` - время начала рабочего промежутка
- `end` - время окончания рабочего промежутка
- `_descr` - комментарий (Здесь я пишу причину почему я вынужден отвлечься в конце промежутка, это будет использоваться для наглядности при анализе эффективности)

## Анализ эффективнности рабочего времени

Результат анализа - это набор графиков для сравнения разных режимов работы.

Заложены следующие правила:
- Минимальный промежуток ля продуктивной работы - 90 мин
- Критический промежуток рабочего времени, который можно считать непродуктивным - 45 мин

::: info
Почему приняты именно такие значения?

Об этом писали на хабре (ссылку приложу позже)
:::

::: warning
Правила заданы как константы для передачи в функцию `getBusinessDayAnalyze`

```ts {5-6}
const analysis = getBusinessDayAnalyze({
  day: (day as EDayEnumValues),
  dayBusinessTime: businessTimeConfig[businessType].cfg[day],
  criteries: {
    goodDiffInMinutes: 90,
    criticalDiffInMinutes: 45,

    // -- NOTE: Пока не используется
    optimalCoffeBreakDiffInMinutes: [10, 15],
    optimalLunchDiffInMinutes: [45, 59],
    // --
  },
})
```
:::
