## Ключевые изменения

- Параметр time: requestAnimationFrame автоматически прокидывает в колбэк высокоточное время (timestamp). Мы используем его для сравнения с `lastUpdateRef`.
- Стабильность: Даже если стейт React "тормозит" из-за тяжелого UI, цикл rAF и отрисовка на Canvas будут стараться работать независимо и плавно.
- Экономия: `setMemState` теперь срабатывает в 60 раз реже, что критически важно для производительности страницы, которую этот же виджет и мониторит.

## By the way

Поскольку `window.performance.memory` обновляется в браузере не каждые 16мс (частота rAF),
а значительно реже (раз в секунду или около того),
вы можете добавить проверку времени внутри `draw`,
чтобы выполнять `setMemState` (тяжелая операция для React) только раз в 500-1000 мс,
при этом оставляя отрисовку графиков плавной.

Для реализации троттлинга мы разделим отрисовку на Canvas (плавная, через rAF) и обновление стейта React (редкое, через проверку времени).
Это позволит графику двигаться плавно, но спасет React от 60 тяжелых ререндеров в секунду.
1. Добавляем `lastUpdateRef`, чтобы следить за временем последнего обновления стейта.
2. В цикле rAF проверяем: если прошло больше 1000мс (или другого интервала), вызываем `setMemState`.
3. Отрисовка на Canvas при этом продолжает работать на максимальной частоте.
